# ComfyNexus 打包说明

## 快速打包脚本使用指南

### 基本用法

```powershell
# 交互式打包（推荐）
.\quick_build.ps1
```

脚本会依次询问：
1. **打包模式**: 生产模式（隐藏控制台）或调试模式（显示控制台）
2. **前端构建**: 是否跳过前端构建
3. **版本号**: 是否指定版本号（默认从 VERSION 文件读取）

### 打包模式说明

#### 🟢 生产模式（默认）
- **特点**: 隐藏控制台窗口
- **适用场景**: 正式发布给用户
- **优点**: 界面更加专业美观，用户体验好
- **缺点**: 无法看到控制台输出，不便于调试
- **选择**: 在交互式界面中输入 `1`

#### 🟡 调试模式
- **特点**: 显示控制台窗口
- **适用场景**: 开发测试、问题排查
- **优点**: 可以看到所有日志和错误信息
- **缺点**: 会显示黑色控制台窗口
- **选择**: 在交互式界面中输入 `2`

### 常见使用场景

#### 场景 1: 日常开发测试
运行 `.\quick_build.ps1`，然后：
1. 选择 `[2]` 调试模式
2. 选择 `[Y]` 跳过前端构建（如果前端未改动）
3. 直接回车使用默认版本号

#### 场景 2: 正式发布
运行 `.\quick_build.ps1`，然后：
1. 选择 `[1]` 生产模式（默认）
2. 选择 `[N]` 完整构建前端
3. 输入版本号，例如 `1.0.2`

#### 场景 3: 快速重新打包（前端未改动）
运行 `.\quick_build.ps1`，然后：
1. 选择 `[1]` 生产模式
2. 选择 `[Y]` 跳过前端构建
3. 直接回车使用默认版本号

#### 场景 4: 调试打包问题
运行 `.\quick_build.ps1`，然后：
1. 选择 `[2]` 调试模式
2. 选择 `[Y]` 跳过前端构建
3. 直接回车使用默认版本号

### 技术细节

#### Spec 文件
- **build_exe.spec**: 生产模式配置（`console=False`）
- **build_exe_debug.spec**: 调试模式配置（`console=True`）

#### 自动选择逻辑
脚本会根据 `-ShowConsole` 参数自动选择对应的 spec 文件：
```powershell
if ($ShowConsole) {
    $SpecFile = "build_exe_debug.spec"
} else {
    $SpecFile = "build_exe.spec"
}
```

### 注意事项

1. **首次打包**: 需要完整构建前端，不要使用 `-SkipFrontend`
2. **版本号**: 建议在正式发布时明确指定版本号
3. **调试模式**: 仅用于开发测试，不要发布给用户
4. **虚拟环境**: 确保已激活 Python 虚拟环境（`.venv`）
5. **依赖检查**: 脚本会自动检查必要的依赖（Node.js、PyInstaller 等）

### 故障排查

#### 问题 1: 打包失败
```powershell
# 使用调试模式查看详细错误信息
.\quick_build.ps1 -ShowConsole
```

#### 问题 2: 前端构建失败
```powershell
# 手动构建前端
cd frontend
npm install
npm run build
cd ..

# 然后跳过前端构建
.\quick_build.ps1 -SkipFrontend
```

#### 问题 3: PyInstaller 未安装
```powershell
# 激活虚拟环境并安装
.\.venv\Scripts\Activate.ps1
pip install pyinstaller
```
